Commands

The button component supports executing commands on other elements. Use the command attribute to specify the command to execute and the commandfor attribute to specify the target element. The command should match a public method name of the target element.

If the commandfor attribute is set to parent, the command will be executed on the parent element. If the button is in the slot of another cek-* element, the command will be executed on the element without needing the commandfor to be an ID.

Styles Native Elements

Buttons

You can apply the buttonStyles to native HTML <button> or <anchor> elements to achieve the same styling as the cek-button component. This is particularly useful in scenarios where you need to use native buttons or anchors for better accessibility or specific functionality.

Not all cek-button functionality is CSS based including loading text, disabled states and commands. The disabled attribute will still work to style the button.

Accessibility

Ensuring accessibility is crucial for providing a good user experience for all users, including those using assistive technologies.

Disabled

Disabled buttons will are still focusable but click events blocked and aria-disabled="true" added.

Loading Text

The loading-text attribute is essential for conveying the loading state to users. When loading is true aria-busy will be set on the button. This is only functional with the cek-button component and does not work with native buttons styled using cek-button styles.

If user's motion preferences are set to prefers reduced motion, the spinner will be hidden. It is recommended to use loading-text in these scenarios to provide clear feedback.

If you are using a native button and still want to display loading feedback, you can use JavaScript to swap the button's default slot content to custom text during the loading state.

Attributes

Attribute Description
type Sets the button type. Possible values are button, submit, and reset. Defaults to button.
disabled Disables the button when present.
loading Shows a loading indicator when present.
arrow Displays an arrow icon when present.
variant Sets the styling variant of the button. Possible values are primary, secondary, outline, ghost, link, destructive, neutral.
size Sets the size of the button. Possible values are small, medium, and large.
icon-only If present, styles the button for icon-only display without text.
pill If present, styles the button with a pill shape.
circle If present, styles the button with a circle shape.
button-aria-label Sets an ARIA label for accessibility.
command The command to execute when the button is clicked. The command should match a public method name of the target element.
commandfor The ID of the element to execute the command on. If set to parent, the command will be executed on the parent element.

Slots

Slot Name Description
(default) The text content of the button.
start Content to display before the button's text (e.g., icons).
end Content to display after the button's text (e.g., icons).
